# テスト設計書: AI学習レコメンド機能

## 1. 概要

本設計書は、テスト計画書（`test-plan.md`）で定義したテスト戦略に基づき、具体的なテスト観点とテストケースを記述する。

### 1.1 対象範囲

- E2Eテスト（Playwright + モック）
- AI出力品質テスト（Python + 実API）

### 1.2 テスト設計の方針

- 入力の組み合わせは同値分割・境界値分析により代表ケースに絞る
- 各テストケースには優先度（高/中/低）を付与する（判定基準はテスト計画書4.5参照）
- 各テストケースには観点IDと対応ACを明示し、トレーサビリティを確保する

## 5. ゴールデンテストセット

AI出力品質テストで使用する評価用入力データ。実務ではPdMおよびコンテンツチームと合意する想定だが、本ポートフォリオでは学習指導要領に基づき自前で定義する。

### 5.1 ゴールデンテストセットの構造

各テストケースは以下の要素を持つ。

- 入力: 学年、理解済み単元、苦手単元
- 期待される推薦単元（複数可）
- 期待される推薦理由の要素（キーワード、言及すべき内容）
- 禁止事項（含まれてはいけない単元名、情報）

### 5.2 サンプル

```json
{
  "id": "GT-001",
  "input": {
    "grade": "中2",
    "understood": ["一次関数"],
    "weak": ["連立方程式"]
  },
  "expected_units": ["連立方程式", "比例"],
  "expected_reason_keywords": ["連立方程式", "一次関数"],
  "forbidden_units": ["微分", "積分", "三平方の定理"],
  "forbidden_keywords": ["存在しない架空の単元名"]
}
```

### 5.3 ゴールデンテストセット一覧

| ID | 学年 | 理解済み | 苦手 | 用途 |
|---|---|---|---|---|
| GT-001 | 中2 | 一次関数 | 連立方程式 | 正確性テスト、一貫性テスト |
| GT-002 | 中3 | 一次関数、連立方程式 | 二次関数 | 正確性テスト |
| GT-003 | 中1 | なし | なし | 境界値（初学者） |
| GT-004 | 高2 | 数と式、二次関数 | 微分 | 高学年での推薦理由検証 |
| GT-005 | 中3 | 比例のみ | なし | 前提知識2段階の検証 |

完全なデータは `tests/ai-quality/golden_test_set.json` に格納する。

## 補足: 単元名表記について

本文中の単元名は一部 `src/units.js` の正式表記と異なる
(例: 「正負の数」→「正の数と負の数」、「比例」→「比例・反比例」)。
`tests/e2e/fixtures/testdata.ts` の `unitMapping` で表記差を吸収済み。
設計書側の用語統一は将来の整理課題として残す。
